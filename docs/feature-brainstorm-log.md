# 機能改善ブレスト記録

## 2026-07-17

### 結論

現行の静的 PWA・JSON 駆動・GitHub Pages 運用を維持する前提で、参加者向け機能とアプリ全体の改善案を検討した。
多角的に案を出したものの、現時点で実装候補として優先するほど有意義な案は見いだせなかった。
以下は採用予定やロードマップではなく、今後の重複検討を避けるための検討履歴である。

### 検討した観点

- 当日支援、意思表明、アンケート、質問、投票
- 端末内の予定・参加履歴・メモ・振り返り
- 外部フォーム、地図、カレンダー等との連携
- 会場移動、現在・次の予定、変更情報への到達
- オフライン、通信障害、アクセシビリティ
- 情報の鮮度、更新履歴、リンクの信頼性
- Excel 取込、公開前確認、イベント終了後の運用
- 匿名の利用計測と改善判断

### 主な案

#### 参加者の操作・記録

- セッションのお気に入り、マイタイムテーブル、次の予定
- 参加済みマーク、セッション別メモ、3欄振り返り
- 資料・持ち物チェックリスト、端末内データ管理
- 事後アンケート、セッション評価、質問投稿、理解度投票
- 交流希望登録、カレンダー追加、地図・経路案内
- 混雑表示、ライブ投票、プッシュ通知、匿名 Q&A

#### 情報設計・運用改善

- 開催前・開催中・終了後の表示切替
- 現在進行中・次の予定・変更情報の優先表示
- セッション検索、日付・会場・登壇者による絞り込み
- 会場間の移動時間、受付・トイレ・休憩所等の案内
- 最終更新日時、更新履歴、情報の有効期間
- 資料の公開状態、外部リンク種別、情報出典の表示
- オフライン時の最終取得日時、事前保存、軽量表示
- 文字サイズ、高コントラスト、読み上げ、印刷表示
- Excel 取込プレビュー、公開前チェック、変更差分要約
- 終了後モード、資料アーカイブ、次回イベントへの導線

### 再検討する場合の条件

今後、具体的な参加者の困りごと、運営上の反復作業、問い合わせ実績、アンケート結果などが得られた場合に限り、その課題から逆算して再検討する。アイデアの多さ自体を目的にせず、実際の利用場面と効果を確認できる小さな改善を優先する。

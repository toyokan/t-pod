# 新イベント追加フロー

申込ページ・チラシ・公開プログラムを生成AIに読ませ、`イベント情報入力シート.xlsx` へ転記し、人が目視修正した後、エージェントがJSONへ変換する手順です。

このExcelは、イベントごとの形式差をAIの推測で吸収せず、表の行とキーで明示するための中間データです。JSONの構造は実装済みの `events/*.json` を正とします。

## 標準手順

1. `イベント情報入力シート.xlsx` をイベントごとに複製する。
2. 申込ページURL、公開チラシ、公開プログラムを生成AIへ渡し、Excelへ入力させる。
3. 人がExcelを目視し、正式名称、日付、時刻、会場、登壇者、URL、資料間の不一致を修正する。
4. 修正済みExcelをエージェントへ渡し、イベントJSON・一覧エントリ・PWA関連ファイルを生成させる。
5. JSON妥当性と画面表示を確認する。

### Excel入力を依頼するプロンプト

> 申込ページ、チラシ、公開プログラムを読み、添付の `イベント情報入力シート.xlsx` に入力してください。資料にない内容は推測せず、「要確認」シートに記録してください。入力例は作らず、対象イベントの情報だけを入力してください。セッションとプログラム項目は、同じセッションキーで対応づけてください。出力は入力済みExcelだけにしてください。

### JSON生成を依頼するプロンプト

> このExcelから `events/<id>.json` を生成し、`events.json` に一覧エントリを追記してください。あわせて `events/<id>.webmanifest` と `assets/icon-<id>.svg` を既存イベントに合わせて生成してください。資料にない内容は推測せず、「要確認」シートに未解決事項があれば作業を止めて報告してください。既存イベントは上書き・削除しないでください。生成後にJSON妥当性とローカル表示を確認してください。

## シート構成

| シート | 役割 | 目視確認のポイント |
| --- | --- | --- |
| `入力ガイド` | 作業順・入力状態の定義 | 空欄と要確認の使い分け |
| `参照元` | AIへ渡した公開資料 | URL、ファイル名、参照範囲 |
| `基本情報` | `eventInfo` と `events.json` | 正式名称、表示用日付、ID、ブランド色 |
| `開催日` | `eventInfo.dates[]` | `day1` 等の日付ID、日付、曜日、表示時間 |
| `会場` | `rooms[]` | 内部の会場IDと画面の表示名を混同しない |
| `セッション` | `sessions[]` の日時・区分 | 1行=1セッション。終了時刻なしは空欄 |
| `プログラム項目` | `sessions[].items[]` | セッションキー、項目順、会場ID、演題、登壇者 |
| `資料リンク` | `venue.resourceLinks[]` / `forms[]` | 公開URLであること |
| `関連書籍` | `books[]` | 書名・著者・URLの対応 |
| `概要・お知らせ` | `eventInfo.notices[]` | 原資料にある案内だけを記載 |
| `要確認` | 不一致・判読不能・不足 | JSON生成前に全件解決する |

## 形式差を吸収するルール

### 日数・会場数・任意セクション

- 開催日、会場、リンク、書籍、お知らせは必要な行数だけ使う。
- 並行会場がないイベントは `会場` シートを空欄のままにし、JSONの `rooms` は空配列または省略可能とする。
- 書籍・資料・お知らせがない場合は各シートを空欄のままにし、架空の項目を補わない。

### セッションとプログラム項目

- `セッション` は1行=1セッションとし、`セッションキー` を一意にする。既存形式に合わせ `d1-01`、`d2-03` のように採番してよい。
- `プログラム項目` は、対応する `セッションキー` を記入して紐づける。
- 同一セッションに複数の演題・会場がある場合、`プログラム項目` を複数行にし、`項目順` を1、2、3…と付ける。
- 受付・休憩・閉会など `items: []` のセッションは `セッション` にだけ記入し、`プログラム項目` には行を作らない。
- 会場なしの複数項目は、会場IDを空欄にしたまま複数行にする。
- `登壇者など` はセル内改行で1行=1要素とし、JSONでは `meta[]` に分割する。
- `控えめ表示=true` の行だけ `items[].subtle: true` を出力する。通常は省略する。
- 終了時刻や開始時刻が原資料にない場合は空欄とし、推測で時刻を補わない。

### 会場

- `会場ID` はJSON内部とプログラム項目の紐づけに使う短いキー。
- `画面の表示名` は利用者に表示する名称。例: 会場ID `講堂`、表示名 `3階・講堂`。
- 色は `blue` / `blueDeep` / `green` / `greenDeep` / `orange` / `purple` から選ぶ。

## JSON変換仕様

- `基本情報` から `eventInfo` と `events.json` の1エントリを生成する。任意項目の空欄はプロパティを省略する。
- 主催・共催・協力・後援はセル内改行で分割し、複数の場合は配列にする。団体名は推測しない。
- `開催日` は上から `dates[]` にする。日付IDは `セッション` の日付IDと一致させる。
- `会場` は `rooms[]` の `{id, name, color}` にする。
- `セッション` の各行を `sessions[]` にし、セッションキーを `id` とする。
- `プログラム項目` をセッションキーで束ね、項目順で並べて各セッションの `items[]` にする。
- `資料リンク` は種別により `eventInfo.venue.resourceLinks[]` または `eventInfo.forms[]` に振り分ける。
- `関連書籍` は `books[]`、`概要・お知らせ` は `eventInfo.notices[]` にする。
- `events.json` の `theme` は個別JSONと同じ全文にする。
- 実体の `events/<id>.webmanifest` と `assets/icon-<id>.svg` を既存イベントと同じ構造で作る。`eventInfo.manifestPath` も設定する。

## 生成後の確認

PowerShell:

```powershell
python -m json.tool "events/<id>.json" > $null
python -m json.tool "events.json" > $null
python -m http.server 8080
```

ブラウザで次を確認します。

- `/` に新イベントが表示される。
- `/?id=<id>` で日付、時刻、会場、演題、登壇者、資料、書籍が原資料どおり表示される。
- 並行セッションと会場色が正しく対応する。
- 外部リンクが公開状態で開く。
- PWAの表示名、アイコン文字、ブランド色が正しい。
- スマホ幅で横切れや読みにくい折り返しがない。

## 注意事項

- 過去の `events/<id>.json` は削除しない。
- `index.html` にイベント固有文言を書かない。
- 配布資料の実ファイルはリポジトリへ置かず、公開URLを記載する。
- `template/make_template.py` は旧形式の再生成用で、現在のv2シートを上書きするため実行しない。
